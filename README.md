A **production-ready automation testing framework** built using modern tools and best practices to validate real-world e-commerce workflows.

## 📌 What This Project Covers

This framework automates key user journeys in an e-commerce application:

* ✅ Login functionality (positive & negative scenarios)
* ✅ Data-driven testing using Scenario Outline
* ✅ Add to cart & checkout flow (E2E)
* ✅ Error validation (locked user, invalid credentials)
* ✅ Performance testing (`performance_glitch_user`)
* ✅ Parallel execution support

---

## 🧱 Framework Architecture

Designed with scalability and maintainability in mind:

* 🔹 Page Object Model (POM)
* 🔹 Reusable step definitions
* 🔹 Environment-based configuration
* 🔹 Modular test structure
* 🔹 TypeScript for strong typing

## ⚙️ Getting Started

### Install dependencies

```bash
npm install
```

### Run tests

```bash
npm run wdio
```

### Run smoke tests

```bash
npm run wdio:smoke
```

### Run tests with Allure report

```bash
npm run test:allure
```
